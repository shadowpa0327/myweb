Normalization

first normal form (1NF)

  • 所有的attribute 都是atomic 的
    • atomic : 只要這個attribute 不能拆分,便可以說這個attribute 是atomic
    • non-atomic : ex : multivalued , composite

  • how to do first normalization ?
    • 作法：把非atomic 的attribute 拆開
  • some problem that shout be cautious
    • 更新資料時很費時
      • 因為資料的重複性很高

Second Normal Form(2NF)

  • 去除partial functional dependency
  • 滿足1NF 且所有的non-key attribute 均為 full functional dependency on key attribute

Third Normal Form(3NF)

  • definition : a relation is said to be 3NF ↔\leftrightarrow ↔﻿ 不存在transitive dependency that point to non key attribute
  • Another definition :
    • if ∀  (α→β) ∈F+\forall \space \space(\alpha\rightarrow\beta)\space \in F^+ ∀  (α→β) ∈F+﻿ at least one of the following condition holds → 3NF
      1. α →β\alpha \space \rightarrow \beta α →β﻿ is trivial
      1. α\alphaα﻿ is superkey(primary key) of R
      1. Each attribute A in β–α is contained in a candidate key for R.

      (3.其實就是代表，遞移相依性只到的東西一定是某組candidate key 的subset)

    1. 拉FK

  • 說明 : BCNF 其實就是3NF 的進階版
  • definition
    1. ∀ α→β∈F+ α \forall \space \alpha\rightarrow\beta \in F^+ \space \alpha \space∀ α→β∈F+ α ﻿is a superkey (可以唯一決定一個instance 的attribute set)
    1. α→β\alpha \rightarrow \betaα→β﻿ is trivial

Armstrong's axioms and some derived property

Closure of a set of functional dependencies

  • definition: 通常用 F+F^+F+﻿ 表示代表的是所有可能由F延伸出來的FD之集合。
  • 所有的延伸FD 都可以由上方表列的 ，性質推出。
  • 一共有幾個呢？
    F+=F repeat
    for each functional dependency f in F+
    apply reflexivity and augmentation rules on f add the resulting functional dependencies to F +
    for each pair of functional dependencies f1and f2 in F + if f1 and f2 can be combined using transitivity
    then add the resulting functional dependency to F + until F + does not change any further
    • 先用augmentation 把基礎FK 做延伸
    • 接著在運用transitivity 找出所有遞移的FK
    • 此演算法其實就是把所有可能的FK 爆開來，然後直到到收斂(不再新增)為止
    • 複雜度： O(4^n) 因為 size 為n的的set 一共有2^n個subset , 如果考慮dependency 的左右兩側，可能產生的FD 一共會有 2^n * 2^n 個
